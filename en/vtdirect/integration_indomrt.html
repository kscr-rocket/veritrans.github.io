VT-Direct Integration (Indomaret)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct Indomaret payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



Indomaret is a feature provided by Indomaret where customer can make online payment at indomaret store. The customer only has to come to indomaret cashier and complete the payment. By using this convenience store channel, customer can pay the bill in any indomaret store. Veritrans will send notification in real time for every incoming payment.

To activate Virtual Account method of payment (automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Integration VT-Direct Indomaret

  1. Create Form
  2. Perform Charge Transaction
  3. Display Payment Code
  4. Check Transaction Status

Unlike the other payment methods (Credit Card and Mandiri ClickPay), Indomaret payment process is not executed on your payment page. Customers make payments via cashier Indomaret store nearby.

  • Step 1Create Form
  • Step 2Charge Transaction
  • Step 3Display Payment Code
  • Step 4Transaction Status


Create Form

Indomaret payment form will be made in the file form_vtdirect_indomrt.html. Because most of the transaction process will not be made in the merchant's page, the form that we made will only show the steps of payment and the payment button of the Indomaret. When customer clicks this payment button, charge transaction process will be carried on the next step.

The following is a sample from the file form_vtdirect_indomrt.html: 

<head>
  <title>Indomaret</title>
</head>
<body>
  <h1>Indomaret</h1>
  <b>I want to pay with Indomaret</b>
  <form action="checkout_indomrt.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
</body>


Charge Transaction

To perform charge transaction using Indomaret, the following steps are required:

  • payment_type parameter has to be set as cstore.

  • New JSON object has to be added, with the name cstore, and has the pairing name/value as followed:

    • store → Store name
    • message → Label to be displayed in Indomaret POS

This step describes the Indomaret charge transaction in the file checkout_indomrt.php. To perform charge transaction, we will use Veritrans PHP library. The following is an example of charge transaction implementation in the file checkout_indomrt.php: 

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data that will be sent for charge transaction request with Indomaret.
$transaction_data = array(
  'payment_type' => 'cstore',
  'cstore' => array(
      'store' => "Indomaret",
      'message' => "Buah Mangga"
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the Indomaret data, transaction data, and customer data as the charge transaction parameter. You can add additional data during the charge transaction, such as: Customer address, delivery address, and item information. To add this data, you can check API Reference - API Methods.


Display Payment Code

Veritrans' server will return a response for every charge transaction in JSON form. Veritrans' PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
  "status_code" => "201",
  "status_message" => "Success, CSTORE transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "cstore",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "payment_code => "498112345234",
  "gross_amount" => "145000.00"
);


Two important information details in the JSON response above are the status_code and payment_code. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. Merchant is required to show the payment_code to the customer so they can finish the Indomaret transaction. You can add the following PHP code at the last line of the file checkout_indomrt.php to do so: 

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "<b>Indomaret</b> <br /><br />";
  echo "<b>Please complete Indomaret payment with this payment code:</b> ".$result->payment_code;
  echo "<br /><br />";
  echo "1. You have 24 hours to complete payment before this transaction expired <br />";
  echo "2. If you need help, please contact 021-22334456";

} else {
  //error
  echo "Have error on transaction.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>


Up to this point, the transaction is stored in MAP with pending status. Transaction status will change to success when your customer completes the payment.

To facilitate the payment process, we provide the payment simulator in sandbox Environment. Enter the payment code previously obtained to check whether there is a transaction with the payment code number.

Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct may fail. This will prevent you from getting the payment code number. Veritrans will return an error message if failure occurrs. The reason of failure can be inspected from the status code displayed on the error message. The status codes that generally occur are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.


The complete list of status code can be found at API Reference - Status Code.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next steps.


Transaction Status

When charge transaction is successful, the transaction status is still pending. Customer has to complete the transaction in Indomaret store. When this process is completed, the transaction status will be changed into success. You can check the transaction status change through MAP.
